# Дорожная карта проекта (Roadmap)

## 🎯 Основная цель: От дрона к трактору
Создать Open Source инструмент, который позволяет фермеру загрузить карту с дрона (Mavic 3M), увидеть проблемные зоны и отправить карту задания в трактор.

## ✅ Выполнено (Foundation)
- [x] Базовая архитектура (SPA + REST API).
- [x] Управление границами полей (рисование, импорт Shapefile).
- [x] Учет земельного банка (собственность/аренда).
- [x] Мобильная версия (PWA) для работы в поле.

## 🚧 Текущий этап: Точность и Данные
- [ ] **Точность площадей:** Исправить расчет гектаров (переход с Web Mercator на равновеликие проекции). Критично для расчета доз удобрений.
- [ ] **Интеграция с кадастром:** Стабилизировать WMS слой Словакии (в ветке `feat/cadastral-integration-debug`).
- [ ] **UX:** Заменить нативные `alert/prompt` на красивые модальные окна.

## 🚁 Этап 2: Обработка снимков (Drone Data)
- [ ] **Загрузка GeoTIFF:** Реализовать загрузку больших растровых файлов (ортофотопланов) на сервер.
- [ ] **Визуализация растра:** Отображение GeoTIFF поверх карты Leaflet (используя `georaster-layer-for-leaflet`).
- [ ] **Расчет индексов (Backend):** Использовать `rasterio` + `numpy` для расчета NDVI ((NIR-Red)/(NIR+Red)) из мультиспектральных снимков.
- [ ] **Аналитика:** Гистограмма распределения индекса по полю.

## 🚜 Этап 3: Точное земледелие (Prescription Maps)
- [ ] **Зонирование:** Алгоритм кластеризации (K-Means или Jenks) для разбиения поля на зоны продуктивности (3-5 зон).
- [ ] **Калькулятор норм:** Интерфейс для ввода доз (кг/га) для каждой зоны.
- [ ] **Экспорт:** Генерация Shapefile (`.shp`, `.shx`, `.dbf`) с атрибутами доз, понятными для терминалов John Deere / Trimble.

## Backlog (Будущее)
- [ ] Поддержка ISOXML (стандартный формат обмена данными в агро).
- [ ] Интеграция с John Deere Operations Center API.
- [ ] Мобильное приложение на Flutter.
