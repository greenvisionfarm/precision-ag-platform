## 🔥 Боль пользователя: Обработка снимков с дрона

### Проблема
> "Я не могу сам обработать снимки с дрона для этого я использую dji smart farm. Это долго."

### Решение (приоритеты)

#### 🔴 Приоритет 1: Ортомозаика из отдельных снимков

**Что нужно:**
- Загрузка набора отдельных JPEG/TIFF снимков с дрона
- Автоматическая склейка в ортомозаику (OpenCV/ALICE)
- Построение NDVI карты из мозаики

**Преимущества:**
- ❌ Не нужен DJI Smart Farm
- ❌ Не нужна подписка DJI FlightHub 2
- ✅ Полная автономность
- ✅ Быстрее (локальная обработка)

**Стек:**
- `opencv-python` — склейка снимков
- `opencv-contrib-python` — SIFT/ORB фичи
- `pygdal` — геореференсинг
- `exiftool` — чтение GPS координат из EXIF

**Оценка:** 3-5 дней разработки

---

#### 🟡 Приоритет 2: Поддержка DJI Terra

**Что нужно:**
- Импорт ортомозаик из DJI Terra
- Парсинг выходных файлов (GeoTIFF + метаданные)
- Интеграция с existing NDVI pipeline

**Преимущества:**
- ✅ Использует существующий workflow
- ✅ Высокое качество обработки
- ✅ Поддержка Mavic 3M

**Оценка:** 1-2 дня разработки

---

#### 🟡 Приоритет 3: WebODM интеграция

**Что нужно:**
- Подключение к WebODM серверу (OpenDroneMap)
- Автоматический запуск обработки через API
- Получение готовой ортомозаики

**Преимущества:**
- ✅ Open Source альтернатива
- ✅ 3D модели, облака точек
- ✅ Сообщество и плагины

**Стек:**
- `pyodm` — Python клиент для WebODM

**Оценка:** 2-3 дня разработки

---

#### 🟢 Приоритет 4: Прямая загрузка с дрона (будущее)

**Что нужно:**
- Подключение к дрону по USB/WiFi
- Скачивание снимков напрямую
- Автоматическая обработка

**Стек:**
- `dji-payload-sdk` — интеграция с DJI
- `gphoto2` — управление камерой

**Оценка:** 5-7 дней разработки
