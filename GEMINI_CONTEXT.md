# Контекст проекта для Gemini

Этот файл служит напоминанием о ключевых аспектах проекта, используемых технологиях и принятых подходах.

## Общая информация

*   **Название проекта:** Field Mapper
*   **Тип проекта:** Веб-приложение для работы с геоданными (полями).
*   **Основная цель:** Загрузка, отображение, переименование и удаление географических полей.

## Стек технологий

### Backend
*   **Язык:** Python
*   **Веб-фреймворк:** [Tornado](https://www.tornadoweb.org/en/stable/) (асинхронный)
*   **ORM:** [Peewee](http://docs.peewee-orm.com/) (для работы с SQLite)
*   **База данных:** SQLite (`fields.db`)
*   **Геопространственная обработка:** [GeoPandas](https://geopandas.org/) (для чтения shape-файлов, перепроецирования, расчета площади)
*   **Формат геометрии:** Well-Known Text (WKT) для хранения в БД.

### Frontend
*   **Языки:** HTML, CSS, JavaScript
*   **Библиотеки:**
    *   [jQuery](https://jquery.com/) (для DOM-манипуляций и AJAX)
    *   [Leaflet.js](https://leafletjs.com/) (интерактивные карты)
    *   [Leaflet.draw](https://github.com/Leaflet/Leaflet.draw) (плагин для рисования/редактирования на карте)
    *   [DataTables](https://datatables.net/) (интерактивные таблицы)
*   **Диалоговые окна:** Нативные `alert()`, `prompt()`, `confirm()` (была попытка внедрить кастомные, но отменена).

## Функциональность

*   **Загрузка:** Принимает ZIP-архивы с shape-файлами, парсит их, перепроецирует в WGS84, рассчитывает площадь и сохраняет поля в БД.
*   **Отображение на карте:** Поля из БД отображаются на интерактивной карте Leaflet.
*   **Отображение в таблице:** Список полей с ID, названием, площадью и другими свойствами.
*   **Удаление полей:** Через API (`DELETE /api/field/delete/<id>`) и UI (кнопка на карте, кнопка в таблице).
*   **Переименование полей:** Через API (`PUT /api/field/rename/<id>`) и UI (кнопка в таблице).
*   **Определение имени поля:** При загрузке поля имя определяется по приоритету: `Field_Name` > `name` > `NAME` > `ID`.

## Тестирование

*   **Фреймворк:** [Pytest](https://docs.pytest.org/en/stable/)
*   **Асинхронные тесты:** [pytest-asyncio](https://pytest-asyncio.readthedocs.io/en/latest/)
*   **Изоляция БД:** In-memory SQLite для каждого теста.
*   **Тестовые данные:** Фикстуры для создания тестовых GeoJSON и ZIP-архивов shape-файлов.

## Подходы и решения

*   **Изоляция тестов:** Использование in-memory SQLite и фикстур для обеспечения чистоты тестовой среды.
*   **Обработка ошибок:** Логирование на бэкенде, AJAX-обработка ошибок на фронтенде.
*   **Управление слоями карты:** Использование `L.FeatureGroup` (`editableLayers`) как единого источника истины для отображаемых геообъектов.
