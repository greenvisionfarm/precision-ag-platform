# Контекст проекта для Gemini

Этот файл служит напоминанием о ключевых аспектах проекта, используемых технологиях и принятых подходах.

## Общая информация

*   **Название проекта:** Field Mapper
*   **Тип проекта:** Веб-приложение для работы с геоданными (полями).
*   **Основная цель:** Загрузка, отображение, переименование и удаление географических полей, управление владельцами полей.

## Стек технологий

### Backend
*   **Язык:** Python
*   **Веб-фреймворк:** [Tornado](https://www.tornadoweb.org/en/stable/) (асинхронный)
*   **ORM:** [Peewee](http://docs.peewee-orm.com/) (для работы с SQLite)
*   **База данных:** SQLite (`fields.db`)
*   **Модели:** 
    *   `Owner`: Модель владельца (имя).
    *   `Field`: Модель поля (геометрия WKT, свойства JSON, внешний ключ на `Owner`).
*   **Геопространственная обработка:** [GeoPandas](https://geopandas.org/) (для чтения shape-файлов, перепроецирования, расчета площади)

### Frontend
*   **Языки:** HTML, CSS (используются CSS Variables), JavaScript
*   **Библиотеки:**
    *   [jQuery](https://jquery.com/)
    *   [Leaflet.js](https://leafletjs.com/)
    *   [Leaflet.draw](https://github.com/Leaflet/Leaflet.draw)
    *   [DataTables](https://datatables.net/)
*   **Диалоговые окна:** Нативные `alert()`, `confirm()`. Для переименования в таблице используется **инлайн-редактирование** (input вместо prompt).
*   **Темная тема:** Реализована через CSS-переменные и атрибут `data-theme` на уровне `html`. Переключение стиля карты (OSM Light / CartoDB Dark Matter) происходит автоматически. Выбор сохраняется в `localStorage`.

## Функциональность

*   **Загрузка:** Принимает ZIP-архивы с shape-файлами. Кнопка загрузки появляется только после выбора файла.
*   **Рисование:** Возможность ручного создания полигонов (полей) через Leaflet.draw. После создания запрашивается имя (через `prompt`).
*   **Редактирование:** Возможность изменять границы существующих полей на карте. При сохранении площадь поля автоматически пересчитывается на бэкенде.
*   **Отображение на карте:** Поля из БД отображаются на интерактивной карте Leaflet.
*   **Отображение в таблице:** Список полей с ID, названием, площадью и именем владельца.
*   **Удаление полей:** Через API и UI (с подтверждением `confirm`).
*   **Переименование полей:** 
    *   На карте: через `prompt`.
    *   В таблице: через **инлайн-редактирование** (клик по названию -> ввод текста -> Enter/Blur для сохранения).
*   **Управление владельцами:** Отдельная страница для добавления и просмотра владельцев.
*   **Назначение владельца:** Возможность смены владельца поля прямо в таблице. Кнопка «Сохранить» появляется только при изменении выбора.

## Тестирование

*   **Фреймворк:** [Pytest](https://docs.pytest.org/en/stable/)
*   **Асинхронные тесты:** [pytest-asyncio](https://pytest-asyncio.readthedocs.io/en/latest/)
*   **Изоляция БД:** In-memory SQLite для каждого теста.
